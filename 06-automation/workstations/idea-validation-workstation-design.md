# AI 工具站选题验证工作站设计

日期：2026-06-09

## 1. 定位

AI 工具站选题验证工作站是 ToolSite OS 阶段二的第一个工作站。

它不是灵感生成器，而是一个证据驱动的选题漏斗：

```text
20 个候选方向
→ 初筛 5 个
→ 免费工具深验 3 个
→ 推荐 1 个 MVP
→ 输出给建站工作站的结构化 Brief
```

第一版以自用为核心，同时为后续服务化和产品化预留结构：

```text
自用验证版
→ 人工服务包
→ 半自动内部工作站
→ 客户自助工具
```

## 2. 第一版边界

- 不使用付费订阅工具。
- 主流程使用 Google Trends、Google SERP、Autocomplete、People Also Ask、Related Searches。
- Google Keyword Planner 是可选增强步骤，需要 Google Ads 账号，但不要求投放广告。
- 不让 AI 编造搜索量、流量或收入。
- 所有搜索量、趋势、广告价值和收益判断必须标明证据来源和置信度。
- 不直接建站，只输出给下一步建站工作站使用的 MVP Brief。

## 3. 高价值领域风险策略

第一版不排除金融、健康、税务等高价值赛道，但必须按风险分层处理。

```text
Green：低风险工具，如 JSON formatter、timestamp converter、image resizer、word counter。
Yellow：高价值但可控工具，如 compound interest calculator、mortgage payment estimator、calorie calculator、salary hourly calculator。
Red：高风险建议工具，如投资推荐、医疗诊断、税务申报建议、法律文件建议、贷款/保险个性化推荐。
```

原则：

- 可以做公式型、估算型、非个性化建议的工具。
- 暂不做影响用户重大财务、健康、法律决策的建议型工具。
- Yellow 选题需要公式来源、边界说明、免责声明和更高置信度证据。
- 20 个候选中允许包含 4-6 个 Yellow 高价值工具，但最终推荐 MVP 时风险分参与排序。

## 4. 工作流模块

### 4.1 赛道设定

输入：

- 目标地区。
- 目标语言。
- 目标用户。
- 允许赛道。
- 暂缓赛道。
- 商业化目标。

默认设定：

```text
目标地区：美国/英语市场优先
候选结构：通用工具 8 个、跨境/出海工具 8 个、AI/Prompt 工具 4 个
风险策略：Green / Yellow / Red 分层
商业目标：长期 SEO 流量 + 可转服务案例
```

输出：

- 本轮选题约束卡。

### 4.2 候选生成

生成 20 个英文工具站候选。

每个候选必须包含：

- 工具名称。
- 目标用户。
- 触发场景。
- 主关键词。
- 3-5 个长尾词。
- MVP 功能。
- 数据来源。
- 风险等级。
- 可能对标网站类型。
- 变现方式。
- 48 小时上线可能性。

### 4.3 风险分层

把 20 个候选分成 Green、Yellow、Red。

输出：

- 风险等级。
- 风险原因。
- 是否可通过免责声明降低风险。
- 是否建议进入搜索验证。
- 替代选题建议。

### 4.4 搜索词拆解

每个候选至少拆出：

- 主关键词。
- 工具型关键词。
- 长尾关键词。
- 问题型关键词。
- 替代词。
- 地区/平台组合词。
- Google Trends 核心验证词。
- Google SERP 手动观察词。

### 4.5 免费数据采集

主流程：

- Google Trends。
- Google SERP 前两页。
- Autocomplete。
- People Also Ask。
- Related Searches。

可选增强：

- Google Keyword Planner。

每个候选输出一张采集任务卡：

- 要查询的关键词。
- Google Trends 查询设置。
- Google SERP 查询设置。
- 需要记录的对标网站名称。
- 需要记录的对标页面 URL。
- 需要观察的 SERP 弱点。
- 需要记录的广告/商业信号。
- 可选 Keyword Planner 查询词。
- 用户需要反馈给 AI 的最小信息。

### 4.6 商业价值验证

验证阶段必须纳入：

- 对标网站名称。
- 对标页面 URL。
- 对标网站类型。
- 搜索量区间。
- 趋势状态。
- SERP 可切入性。
- 预估广告价值。
- 预估收益潜力。
- 变现方式。
- 证据置信度。

收益和搜索量不输出精确承诺，只输出区间和等级：

```text
搜索量：未知 / 低 / 中 / 高 / Keyword Planner 区间
广告价值：低 / 中 / 高
收益潜力：低 / 中 / 高
证据置信度：低 / 中 / 高
```

### 4.7 加权评分、反验证与淘汰

评分维度不等权。第一版权重按“能否获得搜索流量”和“能否快速做出可信 MVP”优先，而不是平均看待所有维度。

| 维度 | 说明 | 分数 | 权重 |
|---|---|---:|---:|
| 搜索意图 | 用户是否主动搜索这个问题 | 1-5 | 18% |
| SERP 可切入性 | 是否有弱站、小站、过时页面或体验差页面 | 1-5 | 18% |
| 痛点强度 | 用户是否急、烦、重复遇到 | 1-5 | 14% |
| 结果明确度 | 工具能否直接给出答案 | 1-5 | 12% |
| MVP 复杂度 | 48 小时内能否上线 | 1-5 | 12% |
| 风险可控性 | 是否能通过边界说明和免责声明控制风险 | 1-5 | 10% |
| 变现适配度 | 是否适合 AdSense、联盟、服务导流或积分工具 | 1-5 | 10% |
| 维护难度 | 是否需要频繁更新数据或承担版权风险 | 1-5 | 6% |

输出两个分数：

```text
weighted_score = Σ(维度分 * 权重)
confidence_score = 低 / 中 / 高
```

强制反验证：

```text
在进入建站前 24 小时内，重新检查推荐 MVP 的主关键词 SERP。
必须记录：
1. 前 10 名是否出现新工具页；
2. 是否有新广告、AI Overview、聚合工具块；
3. 原定可切入弱点是否仍存在；
4. 是否需要下调 SERP 可切入性或 MVP 推荐等级。
```

评分辨别力规则：

- 如果候选之间加权分差小于 0.25，不允许仅凭分数排序，必须进入人工 tie-break。
- tie-break 顺序：SERP 可切入性 → MVP 复杂度 → 风险可控性 → 商业化适配。
- 如果 20 个候选里 70% 以上集中在同一 0.5 分区间，说明评分失去辨别力，必须重新定义候选池或提高门槛。

Stop-loss 规则：

- 如果 20 个候选中 Red 风险超过 50%，本轮候选池作废，回到赛道设定。
- 如果 5 个初筛候选全部缺少明确 SERP 弱点，不进入深验，重新生成候选。
- 如果 3 个深验候选均低于 3.6 加权分，不推荐 MVP，输出“本轮不建站”结论。
- 如果最终推荐在反验证中 SERP 可切入性下降 2 分以上，暂停建站，回到 3 个深验候选重排。

结论：

```text
保留 / 观察 / 淘汰
```

### 4.8 推荐 MVP Brief

最终从 3 个深验候选中推荐 1 个 MVP。

输出：

- 推荐选题。
- 推荐理由。
- 不选另外 2 个的原因。
- 目标用户。
- 主关键词和长尾词。
- 对标网站与可超越点。
- MVP 功能范围。
- 输入字段。
- 输出结果。
- 页面结构。
- SEO title / description / H1。
- FAQ。
- 风险边界与免责声明。
- 预估商业化路径。
- 给建站工作站的 Brief。

## 6. Prompt 库

### 6.1 赛道设定 Prompt

```text
你是 AI 工具站选题研究员。请基于以下目标生成本轮选题约束卡。

项目目标：用低成本方式验证可持续盈利的英文工具站选题。
目标市场：美国/英语市场优先。
商业目标：SEO 流量、AdSense、联盟营销、服务导流。
候选结构：通用工具 8 个、跨境/出海工具 8 个、AI/Prompt 工具 4 个。
风险策略：Green / Yellow / Red 分层，不排除高价值领域，但排除高风险建议型内容。
工具限制：不使用付费订阅工具，允许 Google Trends、Google SERP、Autocomplete、People Also Ask、Related Searches，可选 Google Keyword Planner。

请输出：
1. 本轮选题边界；
2. 优先赛道；
3. 暂缓赛道；
4. 风险判断原则；
5. 商业价值判断原则。
```

### 6.2 候选生成 Prompt

```text
请生成 20 个英文工具站候选方向。

每个候选必须包含：
工具名称、目标用户、触发场景、主关键词、3-5 个长尾词、MVP 功能、数据来源、风险等级、可能对标网站类型、变现方式、48 小时上线可能性。

要求：
1. 不要只生成泛泛想法；
2. 每个工具必须能用一个网页完成 MVP；
3. Yellow 领域可以包含金融/健康/税务相关计算器，但必须是公式型、估算型、非建议型；
4. Red 领域直接标记为暂不推进；
5. 按“低风险 + 有商业潜力 + 可快速上线”排序。
```

### 6.3 风险分层 Prompt

```text
请对以下 20 个候选做 Green / Yellow / Red 风险分层。

判断标准：
Green：低合规风险，主要是格式化、转换、生成、查询、文本处理。
Yellow：高价值但需要免责声明、公式来源、非建议声明。
Red：涉及个性化投资建议、医疗诊断、法律税务建议、高版权风险或平台规则灰区。

请输出：
1. 风险等级；
2. 风险原因；
3. 是否可通过免责声明降低风险；
4. 是否建议进入搜索验证；
5. 如果不建议，给出替代选题。
```

### 6.4 搜索词拆解 Prompt

```text
请为以下候选工具拆解搜索词。

每个候选输出：
1. 主关键词；
2. 工具型关键词；
3. 长尾关键词；
4. 问题型关键词；
5. 替代词；
6. 地区/平台组合词；
7. 适合 Google Trends 验证的 1-3 个核心词；
8. 适合 Google SERP 手动观察的 3-5 个词。

注意：
不要编造搜索量，只生成待验证关键词。
```

### 6.5 免费数据采集任务卡 Prompt

```text

每张任务卡包含：

7. 需要记录的广告/商业信号；
8. 可选 Keyword Planner 查询词；
9. 用户需要反馈给 AI 的最小信息。

输出格式要适合复制到表格。
```

### 6.6 商业价值验证 Prompt

```text
以下是人工采集到的 Trends、SERP、对标网站和可选 Keyword Planner 信息。
请只基于这些证据做商业价值判断，不要编造搜索量或收入。
```

## 7. AI有风文章的吸收与改造

参考文章：

- https://aiyoufeng.com/ai-websites-make-money

保留：

- 使用 Google Trends 观察趋势。
- 使用 Google Keyword Planner 作为可选搜索量增强。
- 优先寻找长期需求、功能单一、逻辑清晰、可 AI 快速搭建的工具。
- 上线后进入 Search Console、AdSense、内容和外链流程。

改造：

- 不直接以高 RPM 为唯一优先级。
- 不把金融、健康等高价值领域一刀切排除，而是按风险分层。
- 不让 AI 直接猜搜索量。
- 不直接生成 Hostinger Horizons 提示词，而是先生成通用 MVP Brief，后续可交给 Codex、Claude、ChatGPT、Hostinger Horizons 或其他建站工具。

## 8. 商业化演进

### 8.1 人工服务包

服务名称：

```text
工具站选题验证包
```

客户输入：

- 行业。
- 目标地区。
- 目标用户。
- 商业目标。
- 预算和风险偏好。

交付物：

- 20 个候选方向。
- 5 个初筛方向。
- 3 个深验方向。
- 1 个推荐 MVP。
- 对标网站和 SERP 观察。
- 风险评级。
- MVP Brief。

### 8.2 半自动内部工作站

把手动步骤逐步替换为数据源适配器：

```text
manual / screenshot / official_account / api / search_tool
→ 统一字段
→ AI 判断
→ 人工复核
→ 报告输出
```

### 8.3 客户自助工具

客户输入行业和目标市场，系统生成候选池、关键词线索、SERP 弱点、风险评分和 MVP 建议。

该阶段需要处理：

- 账号连接。
- 数据额度。
- 地区搜索差异。
- API 合规。
- 结果可解释性。
- 报告可信度。

## 9. 第一版成功标准

- 能从 20 个候选中稳定筛出 1 个推荐 MVP。
- 推荐理由能被证据链支撑。
- 能避免 AI 编造搜索量和收入。
- 能形成可复用的表格、Prompt 和报告结构。
- 能直接接入下一步建站工作站。
- 能转化成第一版工具站选题验证服务包。

## 10. 插件与 Skill 调用

全局调用地图见：

- `08-knowledge/plugin-skill-map.md`

本节是选题验证工作站的模块级调用规则，只覆盖当前工作站，不代表整个 ToolSite OS 的全部调用策略。

默认调用原则：

- 赛道设定使用 `brainstorming`，先确认目标、边界和方案。
- 外部文章、教程和竞品页面读取优先使用 `anysearch`；不可用时使用浏览器或网页搜索。
- 涉及当前政策、价格、SEO 规则和 Google 工具要求时，必须搜索或浏览验证。
- Google Trends、Keyword Planner、Search Console、AdSense 等需要登录态的环节优先使用 Chrome。
- 评分表、关键词表、商业价值验证表优先使用 Spreadsheets。
- 自用阶段输出 Markdown；客户交付阶段使用 Documents 生成报告。
- 完成关键产出前使用 `verification-before-completion` 做证据检查。
