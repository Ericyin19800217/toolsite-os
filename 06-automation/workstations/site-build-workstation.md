# 建站工作站

## 目标

根据 MVP Brief 生成第一个可运行、可测试、可部署的工具站页面。

## 输入

- MVP Brief。
- 产品设计原则。
- 公式来源。
- SEO 结构。

## 输出

- 可运行站点。
- 核心计算逻辑。
- 单元测试和组件测试。
- 页面元信息。
- 源码提交。

## 步骤

1. 选择技术栈和页面结构。
2. 实现纯函数计算逻辑。
3. 为计算逻辑写红绿测试。
4. 实现交互组件。
5. 为组件写可访问性和约束测试。
6. 实现 Astro 页面和 SEO 元信息。
7. 跑 `npm run test` 和 `npm run build`。

## 质量门槛

- 不能编造公式、搜索量、收益或来源。
- 数字输入 HTML 约束必须与 JS 校验一致。
- 结果变化必须对辅助技术可感知。
- 交互组件不能无条件首屏 hydration，除非首屏核心交互必须立即可用。
