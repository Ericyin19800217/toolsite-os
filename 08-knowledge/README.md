# ToolSite OS 项目知识库

## 定位

这里沉淀 ToolSite OS 专属的可复用认知、原则、模式和经验。

项目知识库不是归档区，也不是日记。它只保存能帮助后续选题、建站、SEO、推广、服务化和产品化持续复用的知识。

## 与 Obsidian 的关系

```text
项目内知识库 = ToolSite OS 的操作系统
Obsidian = 个人长期知识网络
```

使用规则：

- 项目执行中的流程、模板、Prompt、案例和判断，先沉淀在这里。
- 完成关键节点后，再把可跨项目复用的认知二次提炼进 Obsidian。
- Obsidian 不保存全部过程材料，只保存更通用的方法论和个人复盘。
- 本知识库保留证据链、上下文和项目专属决策。

## 项目主入口

工作站相关设计和计划优先沉淀在项目业务目录：

```text
06-automation/workstations/
```

`docs/superpowers/` 只作为 skill 工作流的过程记录，不作为项目主知识入口。

## 更新触发

以下情况应该更新本知识库：

- 形成新的关键判断。
- 否定一个看似合理的方向。
- 完成一次选题验证、上线、SEO、推广或复盘。
- 发现一个可复用 Prompt 或评分规则。
- 形成可收费服务包的新假设。
- 对协作方式、项目推进方式形成新的原则。
