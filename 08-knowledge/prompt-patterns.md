# Prompt 模式

## 证据优先模式

用途：

避免 AI 编造搜索量、收入或竞品结论。

模板：

```text
请只基于以下证据做判断，不要编造搜索量、流量、收入或广告收益。

证据：
- 查询词：
- 目标地区：
- 检查日期：
- Google Trends 观察：
- SERP 前两页观察：
- 对标网站：
- 可选 Keyword Planner 数据：

请输出：
1. 搜索需求：低 / 中 / 高；
2. 搜索趋势：上升 / 稳定 / 下降 / 季节性；
3. SERP 可切入性：强 / 中 / 弱；
4. 广告价值：低 / 中 / 高；
5. 收益潜力：低 / 中 / 高；
6. 置信度：低 / 中 / 高；
7. 是否进入下一步验证。
```

## 多方案比较模式

用途：

任何关键决策前，主动比较替代方案。

模板：

```text
请针对以下决策提出 2-3 个可行方案，并比较它们的短期执行成本、长期复用价值、商业化潜力和风险。

决策：

约束：

请输出：
1. 推荐方案；
2. 不推荐方案及原因；
3. 需要现在预留的未来扩展点；
4. 如果选择错误，后续代价是什么。
```

## MVP Brief 模式

用途：

把已验证选题交给建站工作站。

模板：

```text
请基于已验证选题生成一个工具站 MVP Brief。

必须包含：
1. 工具名称；
2. 目标用户；
3. 主关键词和长尾词；
4. 对标网站与可超越点；
5. 核心输入字段；
6. 输出结果；
7. 页面结构；
8. SEO title / description / H1；
9. FAQ；
10. 风险边界与免责声明；
11. 移动端要求；
12. 验收标准。

不要生成代码，只输出给建站工作站使用的结构化 Brief。
```
