# ToolSite OS 项目总览

创建日期：2026-06-09

## 项目定位

ToolSite OS 是一个围绕 AI 工具站矩阵的个人资产生产与服务产品化项目。

它不是单纯复制 Hostinger，也不是把 AI 建站理解为“广告躺赚”。项目目标是：

```text
自建工具站矩阵
→ 沉淀从选题到上线、SEO、推广、复盘、维护的智能工作流
→ 用自媒体公开复盘形成信任与流量
→ 将跑通的流程产品化为面向小白、AI 服务商、出海企业、跨境电商和个人工作室的服务或工作站
```

## 当前核心共识

1. Hostinger Horizons 已经显著降低了小白建站、部署、绑定域名、局部修改的门槛。
2. Hostinger 尚未充分自动化选题验证、关键词研究、SEO 优化、外链拓展、推广、数据复盘和长期维护。
3. 自建工具站矩阵可以做，但第一阶段不应只以广告收入为成功标准。
4. 第一阶段真正的成功标准是跑通闭环、沉淀工作流、形成内容素材和可复用资产。
5. B/C 端服务不应过早启动成重平台，而应从自用工作流和真实案例中长出来。
6. 自媒体不是最后的包装，而应从 Day 1 开始记录过程，形成分发与信任飞轮。

## 三阶段路线

### 阶段一：个人工具站矩阵实验田

目标：不用 Hostinger 付费订阅，先用低成本技术栈跑通 1-2 个工具站。

建议技术栈：

- Codex / Claude / ChatGPT：需求分析、PRD、代码生成、修改。
- Astro / Next.js / Vite：工具站框架。
- Cloudflare Pages / Vercel / Netlify：免费部署。
- GitHub：版本管理。
- Cloudflare：DNS、域名、缓存、安全。
- Google Search Console：收录与搜索表现。
- Plausible / Umami / Google Analytics：访问分析。
- Playwright / Lighthouse：自动 QA 和性能检查。
- Obsidian：选题、复盘、资产库。

阶段性产出：

- 20 个工具站候选方向。
- 3 个完成验证的选题。
- 1-2 个上线 MVP。
- 1 套选题到上线工作流。
- 1 套 SEO / 推广 / 复盘模板。
- 10-20 条自媒体内容素材。

### 阶段二：小白智能工具站工作流

目标：把阶段一的经验变成半自动工作站，而不是马上做 SaaS。

候选工作站：

- AI 工具站选题工作站。
- AI 工具站建站工作站。
- AI 工具站 SEO 工作站。
- AI 工具站上线体检工作站。
- AI 工具站 30 天复盘工作站。

最小形态可以是：

```text
Obsidian 模板 + Prompt + Codex 流程 + 表格 + QA 清单
```

### 阶段三：B/C 端垂类服务

目标：把自用闭环和工作流转成可收费服务。

潜在客户：

- 中国出海企业。
- 跨境电商团队。
- 个人工作室。
- AI 应用接单者。
- Vibe Coding 服务商。
- 自动化工作室。
- 小白副业玩家。

候选服务：

- 工具站选题验证包。
- 行业小工具 MVP 上线包。
- AI-built 网站上线 QA 包。
- SEO / Search Console / AdSense 合规体检包。
- 月度维护与优化推广包。

## 项目原则

```text
用第一性原理判断方向，用类比思考借鉴 Hostinger。
先自用跑通，再对外服务。
先半自动工作流，再考虑产品化平台。
先做真实工具价值，再考虑广告收益。
先记录过程，再期待自媒体转化。
```
