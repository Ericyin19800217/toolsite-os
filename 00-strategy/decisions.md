# 阶段性决策记录

日期：2026-06-09

## D1：不直接复制 Hostinger

Hostinger 的强项是基础设施和小白体验。ToolSite OS 的机会在前后两端：

- 前端：选题、关键词、SERP、需求判断。
- 后端：SEO、推广、复盘、维护、服务化。

## D2：允许自建工具站矩阵

“工具站广告变现不适合第一现金流”是风险提示，不是禁令。

当前允许把自建工具站矩阵作为：

- 实验田；
- 内容素材；
- 工作流训练场；
- 服务化案例来源；
- 长期资产池。

## D3：暂不购买 Hostinger

第一阶段验证的是工作流能力，不是 Hostinger 订阅价值。

优先使用：

- Codex / Claude / ChatGPT；
- Astro / Next.js / Vite；
- Cloudflare Pages / Vercel / Netlify；
- GitHub；
- Search Console；
- Playwright / Lighthouse。

## D4：自媒体从 Day 1 开始

自媒体是验证过程的一部分，而不是项目成功后的宣传。

内容主线：

```text
小白也能看懂的 AI 工具站矩阵实战
```

## D5：第一批避开高合规风险

不碰：

- 金融投资建议；
- 医疗健康建议；
- 法律建议；
- 税务专业建议；
- 高版权风险内容。

优先：

- 英文单词工具；
- 图片尺寸/格式工具；
- 时间戳/日期计算器；
- JSON / 文本处理工具；
- Prompt / 模板生成器；
- 跨境卖家小工具；
- AI 工具对比/导航。
