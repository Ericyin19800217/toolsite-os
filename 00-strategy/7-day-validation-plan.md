# 7 天最小验证计划

目标：

```text
不用 Hostinger，跑通 1 个低风险工具站从选题到上线的完整闭环。
```

## Day 1：选题池

- 产出 20 个候选工具站方向。
- 每个方向记录目标用户、搜索词、用户意图、MVP 功能、数据来源、合规风险、变现方式。

## Day 2：筛选 3 个方向

验证：

- 首页是否被大站完全霸榜；
- 是否有新站、小站、弱站进入前两页；
- 是否存在长尾词；
- 用户是否需要工具，而不是只需要文章；
- 是否存在合规风险。

## Day 3：写第一个 MVP PRD

必须包含：

- 首页结构；
- 核心工具功能；
- 输入输出字段；
- SEO title / description / H1 / FAQ；
- About / Contact / Privacy / Terms / Disclaimer；
- sitemap.xml / robots.txt；
- 移动端要求；
- 验收标准。

## Day 4：生成和部署

建议路线：

```text
Codex 生成静态工具站
→ GitHub 管理
→ Cloudflare Pages / Vercel / Netlify 部署
→ 绑定域名或使用临时域名
```

## Day 5：Search Console 与基础 SEO

- 添加 Search Console。
- 提交 sitemap。
- 检查 title / description。
- 检查 canonical。
- 写 1-3 个长尾说明页或 FAQ 页。

## Day 6：推广与反馈

- 提交到相关工具目录。
- 发 build-in-public 复盘。
- 找 3-5 个真实用户试用。
- 不做自动评论 spam。
- 不买垃圾外链。

## Day 7：复盘

判断：

- 是否完成上线；
- 哪一步最卡；
- 是否形成可复用流程；
- 是否能写出 3 条内容；
- 是否有用户反馈；
- Search Console 是否能正常提交；
- 下一站是否可以复用 50% 以上流程。
