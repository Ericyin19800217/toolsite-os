# ToolSite OS 项目 Brief

## 背景

Hostinger Horizons 证明：AI 已经能让小白通过自然语言生成网站、局部修改、发布上线、绑定域名，并接入 Google AdSense。

但 Hostinger 仍没有充分自动化以下环节：

- 工具站选题；
- 关键词与竞争验证；
- SERP 弱点分析；
- SEO 页面结构；
- Search Console 提交与复盘；
- 外链与推广动作；
- 长期维护；
- B/C 端垂类服务包装。

## 项目目标

设计并验证一套比 Hostinger 更完整的工具站生产工作流：

```text
选题 → 验证 → PRD → 建站 → 部署 → QA → SEO → 推广 → 复盘 → 维护 → 服务化
```

## 第一阶段目标

30 天内跑通个人工具站矩阵实验田：

- 20 个候选工具站方向；
- 3 个完成验证的选题；
- 1-2 个上线工具站 MVP；
- 1 套可复用工作流；
- 1 套 SEO / 推广 / 复盘模板；
- 10-20 条自媒体内容素材。

## 长期目标

形成面向以下客户的工作站/服务包：

- 纯小白副业玩家；
- AI 应用接单者；
- Vibe Coding 服务商；
- 中国出海企业；
- 跨境电商；
- 个人工作室；
- 自动化工作室。

## 项目原则

```text
用第一性原理判断方向，用类比思考借鉴 Hostinger。
先自用跑通，再对外服务。
先半自动工作流，再考虑产品化平台。
先做真实工具价值，再考虑广告收益。
先记录过程，再期待自媒体转化。
```
